## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
The milestones generated by the milestone-helper.sh script are not fully compatible with the backlog web UI. Currently, milestones created via the script cannot be selected or used in the web interface. This task is to investigate and fix the milestone creation process so that milestones are fully recognized and selectable in the backlog web UI. Ensure the fix is robust and update documentation if needed.
<!-- SECTION:DESCRIPTION:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. **Root cause confirmed**: The backlog web UI identifies milestones via the `name:` frontmatter field. The milestone-helper.sh script writes `title:` instead of `name:`, so script-created milestones are invisible/unselectable in the web UI.

2. **Fix `cmd_create_milestone()` in milestone-helper.sh**: Replace the `title:` YAML key with `name:` in the heredoc that generates the milestone file. The generated frontmatter should become:
   ```yaml
   ---
   id: milestone-N
   name: "<title>"
   description: "<description>"
   status: active
   created_date: YYYY-MM-DD
   ---
   ```
   Keep all other fields (`id`, `description`, `status`, `created_date`) as they provide useful metadata, just change `title:` → `name:`.

3. **Fix `cmd_assign_task()` lookup in milestone-helper.sh**: The assign-task subcommand currently sets `milestone: <milestone_title>` in task frontmatter by matching on the literal title string provided by the user. This logic is independent of the frontmatter key used in milestone files, so no change needed here — but verify the task frontmatter key matches what the web UI expects when filtering tasks by milestone.

4. **Backfill existing script-created milestones**: The four existing script-generated milestone files (`milestone-1` through `milestone-4`) all have `title:` instead of `name:`. Each must be updated by replacing `title:` with `name:` in their frontmatter. Do this with sed or direct edit:
   - `backlog/milestones/milestone-1 - Create Backlog Skill.md`
   - `backlog/milestones/milestone-2 - Security-Agent-and-Agent-Pipeline-Improvements.md`
   - `backlog/milestones/milestone-3 - Backlog-CLI-Milestone-Helper.md`
   - `backlog/milestones/milestone-4 - backlog-cli-skill-cleanup.md`
   Use: `sed -i 's/^title:/name:/' <file>` for each.\n\n5. **Update script documentation/comments**: Update the inline comments in `milestone-helper.sh` (especially the YAML block example in the file header/usage if present) to reflect that `name:` is the correct field.\n\n6. **Update SKILL.md if it documents the milestone frontmatter format**: Search `backlog/docs/` and the skill SKILL.md for any frontmatter examples using `title:` and correct them to `name:`.\n\n7. **Test AC #3**: After the fix, run the script to create a new test milestone and verify it appears and is selectable in the backlog web UI. Also verify existing milestones (after backfill) are selectable.\n\n8. **Files to modify**:\n   - `.github/skills/backlog-cli/scripts/milestone-helper.sh` — change `title:` → `name:` in heredoc\n   - `backlog/milestones/milestone-1 - Create Backlog Skill.md` — patch frontmatter\n   - `backlog/milestones/milestone-2 - Security-Agent-and-Agent-Pipeline-Improvements.md` — patch frontmatter\n   - `backlog/milestones/milestone-3 - Backlog-CLI-Milestone-Helper.md` — patch frontmatter\n   - `backlog/milestones/milestone-4 - backlog-cli-skill-cleanup.md` — patch frontmatter\n   - SKILL.md (if it references frontmatter format) — update examples\n\n9. **Commit all changes** to satisfy DoD #1.
